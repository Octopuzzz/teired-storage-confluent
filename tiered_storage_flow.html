// ACTIVE FLOW INFO
Write Flow — Producer ke Leader
Producer mengirim events ke Leader Replica menggunakan Kafka Producer API biasa. Producer tidak perlu dikonfigurasi khusus — tiered storage sepenuhnya transparan. Follower replicas menarik data dari leader untuk replikasi.
// TIERING PROCESS — 5 LANGKAH
01
Segment Roll & Flush
Active segment penuh/expired → di-roll (ditutup) dan di-flush ke disk. Kini eligible untuk tiering.
02
Leader Upload Segment
Partition leader upload segment file ke remote object store (MinIO/Ceph) via storage plugin API.
03
Tulis Metadata
Leader mencatat referensi lokasi object di remote ke internal topic _confluent-tier-state.
04
Follower Sync Metadata
Follower fetch metadata dari _confluent-tier-state dan update in-memory map mereka.
05
Hapus Local Segment
Setelah hotset.ms expired, segment lokal dihapus. Data tetap aman di remote object store.
// KONFIGURASI ON-PREMISE (server.properties)
# Enable Tiered Storage
confluent.tier.enable=true
 
# Backend: S3-compatible (MinIO pakai ini)
confluent.tier.backend=S3
confluent.tier.s3.bucket=kafka-tiered-storage
confluent.tier.s3.region=us-east-1
 
# Override endpoint ke MinIO on-prem
confluent.tier.s3.endpoint.override=http://minio.internal:9000
 
# Berapa lama data tetap di local (hotset)
confluent.tier.local.hotset.ms=3600000 # 1 jam
 
# Per-topic: aktifkan tiering
confluent.tier.enable=true # di topic config
segment.bytes=536870912 # 512MB per segment
