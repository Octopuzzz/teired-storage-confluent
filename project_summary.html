Tiered Storage
Project Summary

Ringkasan arsitektur, komponen, dan konsep kunci dari implementasi Confluent Tiered Storage on-premise dengan MinIO sebagai remote object store.

📥 Download Presentasi (.pptx) ⚡ Lihat Flow Diagram
// LATAR BELAKANG
😰
Masalah: Kafka Tradisional
  • Biaya disk premium (NVMe/SSD) sangat tinggi untuk retensi data jangka panjang.
  • Disk broker harus di-scale seiring pertumbuhan data — tidak elastis.
  • Consumer data lama bersaing thread yang sama dengan RT consumer (noisy neighbor).
  • Retensi biasanya dibatasi 7–30 hari — data harus dibuang pemanen.
  • Scale storage harus selalu dibarengi scale compute — tidak bisa pisah.
🚀
Solusi: Tiered Storage
  • Data lama otomatis dipindah ke object store (MinIO) yang jauh lebih murah.
  • Disk broker hanya perlu menyimpan hotset — data yang sering diakses saja.
  • Historical consumer pakai dedicated thread — RT consumer tidak pernah terganggu.
  • Retensi data praktis unlimited — MinIO bisa scale storage secara independen.
  • Hemat hingga ~90% biaya storage untuk topik dengan retensi panjang.

// KOMPONEN UTAMA
👑
Partition Leader
Satu-satunya broker yang melakukan upload segment ke MinIO. Menulis metadata ke _confluent-tier-state setelah upload sukses.
WRITE + TIER
🔁
Follower Replicas
Mereplikasi data dari leader via ISR. Sync metadata tier dari _confluent-tier-state. Siap jadi leader baru jika failover.
REPLICATION
📋
_confluent-tier-state
Internal Kafka topic yang menyimpan mapping antara partition offset dan lokasi object di MinIO. Backbone dari tiered storage awareness.
METADATA
🗄️
MinIO Object Store
Backend penyimpanan cold tier yang kompatibel dengan S3 API. Di-deploy on-premise. Menyimpan segment dalam format identik dengan local.
COLD STORAGE
🧵
Tier Fetch Thread
Dedicated thread pool khusus untuk stream data dari MinIO ke consumer. Sepenuhnya terpisah dari network thread RT consumer — zero contention.
ISOLATION
⏱️
Hotset & Segment Roll
Segment di-roll saat mencapai segment.bytes atau segment.ms. Setelah terupload dan hotset.ms berlalu, local copy dihapus secara aman.
